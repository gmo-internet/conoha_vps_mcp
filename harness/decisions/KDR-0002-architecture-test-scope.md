# KDR-0002: L4構造テスト対象ルールの選定

- 起源: エスカレーションラダー導入 (KDR-0001)
- レベル: L4
- 状態: 有効

## 経緯

L4（構造テスト）に昇格するルールの選定基準を定める必要がある。全31ルールをL4にすると過度なテストになるため、アーキテクチャ不変条件に限定する。

## 決定

以下の10ルールをL4対象とする:

- **A-2**: テストファイル同一ディレクトリ配置
- **A-3/H-1**: ファイル名kebab-case
- **C-1**: `z.object()` に `.strict()` 付与（`*-schema.ts` のみ、`index.ts` のMCP inputSchemaは除外）
- **E-5/E-6/G-4**: テストファイルのインポート・モックパスに `.js` なし
- **F-1**: `@packageDocumentation` 必須
- **G-1**: ソースファイルの相対インポートに `.js` 拡張子
- **H-2**: エクスポート関数名camelCase

### 選定理由

- ファイルシステム構造で機械的に検証可能
- 正規表現ベースの静的チェックで十分な精度が得られる
- 違反するとビルドや型安全性に直接影響する

## 執行方法

`src/architecture.test.ts` にVitestテストケースとして実装。`npm test` で自動実行。
