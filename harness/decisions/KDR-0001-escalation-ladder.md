# KDR-0001: エスカレーションラダー導入

- 起源: レビュー知見の体系化
- レベル: L1
- 状態: 有効

## 経緯

`CODING_PATTERN.md` にパターンを記述するだけでは、違反の検出がPRレビュー時の人間の注意力に依存していた。パターン違反の検出を段階的に自動化し、レビューコストを削減するため、4段階の執行モデルを導入する。

## 決定

パターン執行を以下の4段階に分類する:

1. **L1: ドキュメント** — `CODING_PATTERN.md` および `harness/patterns/` に記述
2. **L2: AIスキルチェック** — `coding-pattern-check` スキルで検出
3. **L3: CIルール** — Biome等のCIツールで自動検出
4. **L4: 構造テスト** — `src/architecture.test.ts` でテスト実行時に自動検出

## 執行方法

- `harness/ESCALATION.md` にレベル定義とルール対応表を記載
- 各パターンファイルの `enforcement-level` フロントマターでレベルを明示
- レベル昇格時はKDRを作成して経緯を記録
